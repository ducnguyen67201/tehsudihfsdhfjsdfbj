# GTM Founder Outreach Message

Use this skill when the user already has a person in mind and wants the message
to sound human.

## When To Load Context

Read these first:

- `business/gtm/icp-and-positioning.md`
- `business/gtm/outreach-copy.md`

Read `.skills/gtm-founder-outreach/SKILL.md` only if the user also wants lead
qualification, ICP fit, or scoring.

## Voice To Aim For

Write like the user:

- nice
- enthusiastic
- respectful
- genuinely curious
- happy to learn
- open to helping, not just asking for help

The message should feel like a real person reaching out, not a growth script.

## Core Rules

- Keep it short.
- Sound warm, not polished-to-death.
- Lead with the real reason for reaching out.
- Use a shared context when available: event, judge panel, mutual friend,
  product, post, podcast, company stage, or specific experience.
- Ask for insight, not a meeting-heavy favor.
- Mention TrustLoop only when it helps the context; do not make the first note
  feel like a disguised pitch.
- Avoid flattery that says nothing.
- Avoid generic lines like "would love to pick your brain".
- Avoid stacking too many asks in one message.

## Default Structure

Most strong messages should follow this order:

1. quick personal opener with a real connection point
2. one sentence on why the person came to mind
3. one sharp question or curiosity
4. soft close that leaves room for a reply

## Message Types

### 1. Cold-but-warm outreach

Use when there is a light connection point.

- open with the shared context
- ask one specific question
- keep product talk minimal

### 2. Insight ask

Use when the goal is learning from their early days.

- focus on what they did, not what the user is selling
- ask about customer feedback, support, ops pain, or founder habits
- make the question easy to answer

### 3. Light follow-up

Use when bumping a prior message.

- keep it very short
- restate the curiosity, not the full story
- never guilt the recipient

## Output Format

Default output:

- `Primary draft`: best version to send
- `Why it works`: one sentence
- `Alt draft`: a slightly warmer or more casual variant

When useful, also include:

- `Short connection request`: under 300 characters
- `Follow-up`: one short bump message

## Writing Heuristics

- Prefer one concrete question over three vague ones.
- If the user says "I want to learn", make that believable by asking about a
  real stage, decision, or behavior.
- If there is a prestige connection, use it lightly. The message should never
  sound like status-borrowing.
- If the note mentions TrustLoop, frame it as context for the curiosity, not as
  the headline.
- If the recipient is not an obvious ICP match, optimize for founder learning,
  not discovery of product pain.

## Good Direction

`Hi Linh — we built TrustLoop at LotusHacks, and I saw you were one of the judges.`

`I’ve been curious about your early LOGIVAN days, especially how hands-on you were with customers at the start.`

`If you’re open to it, I’d love to learn how you approached that phase.`

Why this works:

- specific opener
- genuine curiosity
- no hard pitch

## Bad Direction

`Hi Linh, I am building TrustLoop which automates support and would love to get your advice on my startup.`

Why this fails:

- sounds transactional
- centers the sender too early
- turns curiosity into a disguised pitch

## Completion Checklist

- message sounds like a person, not a template
- asks for insight, not a demo call
- includes a concrete anchor
- product mention is light and earned
- tone is warm, curious, and helpful
